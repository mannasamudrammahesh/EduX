🚨 Fix MongoDB Connection Error

Whitelist Vercel IPs in MongoDB Atlas

Error: MongooseServerSelectionError: Could not connect to any servers in your MongoDB Atlas cluster. One common reason is that you're trying to access the database from an IP that isn't whitelisted.

Follow These Steps:

1

Login to MongoDB Atlas

Open MongoDB Atlas and login with your credentials:

Open MongoDB Atlas →
2

Navigate to Network Access

In the left sidebar, under Security, click Network Access

This is where you manage which IP addresses can connect to your database.

3

Add IP Address

Click the green + ADD IP ADDRESS button in the top right corner.

In the dialog that appears, select "Allow Access from Anywhere"

This will automatically fill in: 0.0.0.0/0

4

Confirm Changes

Click the Confirm button to save your changes.

You should see a new entry in your IP Access List with status "Active" ✓

⏱️ Wait 1-2 Minutes

After adding the IP address, wait 1-2 minutes for the changes to propagate across MongoDB's network. This is normal and required.

5

Test Your Connection

After waiting, test your backend API to verify the fix:

🧪 Test Your Backend

Enter your backend URL:

✅ Success Indicators

  • API responds in less than 2 seconds
  • No timeout errors in Vercel logs
  • Login works without errors
  • No "IP not whitelisted" messages

🔒 Is This Secure?

Yes! While 0.0.0.0/0 allows all IPs, your database still requires username and password authentication. This is the standard practice for serverless deployments like Vercel, which use dynamic IP addresses.
